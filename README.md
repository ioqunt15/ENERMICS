# Design-First AI Harness Foundation

본 프로젝트는 AI 에이전트(Antigravity, Gemini, Claude 등)가 소프트웨어 개발, 디자인, 비즈니스 구축 태스크를 진행할 때 발생할 수 있는 여러 실패 유형(과도한 무작위 수정, 디자인 미개선, 무한 루프 에러, 컨텍스트 용량 초과로 인한 논리 붕괴 등)을 제어하고, 안전하고 구조화된 협업 체계를 유지하기 위한 **범용 AI 개발 하네스 파운데이션**입니다.

## 1. 하네스의 핵심 특징
1. **디자인 우선 게이트 (Design-First Gate)**: 코드 변경 사항이 빌드 및 테스트를 통과하더라도 시각적인 개선이 확인되지 않으면 작업은 완료될 수 없습니다. `Design Evaluator`가 시각적 완성도 점수를 측정하고 통과Verdict를 선언해야 합니다.
2. **역할(Role)과 일꾼(Worker)의 분리**: Claude나 Gemini 등의 AI 모델(Worker)은 고정된 역할이 아니며, 상태 엔진에 기재된 `Current Role`을 기반으로 동적으로 태스크를 수행합니다.
3. **버그 패치 전 반드시 재현 (Reproduce Before Patch)**: 재현할 수 없는 버그는 억측으로 패치하지 않고 상태 추적 및 증적 분석을 강제합니다.
4. **엄격한 컨텍스트 다이어트**: 무분별한 참조 문서 적재를 막고 필요한 단계의 역할 및 스킬 파일만 로드하도록 규칙을 제공합니다.
5. **KISS / YAGNI / DRY 준수**: 무의미한 추상화나 불필요하게 복잡한 기능(스펙성 설계) 생성을 철저히 방지합니다.

## 2. 폴더 아키텍처
* **`agents/`**: 에이전트의 역할별 규범 가이드라인 (Mission, Allowed/Forbidden Actions, Stop Conditions 등).
* **`docs/`**: 프로젝트 계획서, 디자인 방향성, 반응형 레이아웃 및 각종 바이블 역할을 할 산출물 템플릿.
* **`system/`**: 활성 작업자 락(`locks.md`), 상태 엔진(`state.md`), 검증 및 이슈 로그 등이 누적되는 상태 제어 폴더.
* **`workflows/`**: 태스크 시작부터 버그 수정, 반응형 패치, 종료 시까지 에이전트가 밟아야 하는 단계별 흐름서.
* **`skills/`**: 특정 활동(디자인 평가, 코드 사이즈 진단 등)을 완수할 수 있는 세부 지침 및 YAML 프론트매터 스킬 파일.
* **`scripts/`**: 락 상태 검사, 파일 크기 검사, 에이전트 Handoff를 돕기 위한 PowerShell(`.ps1`) 및 Bash(`.sh`) 자동화 스크립트.
* **`prompts/`**: 특정 모델 및 보조 도구가 시작될 때 직접 주입해야 하는 고정 프롬프트 리스트.

## 3. 시작 및 인계 방법
1. 신규 작업을 수행할 때 `system/state.md`를 초기화하고 대상 작업의 요건을 기입합니다.
2. 작업 AI 모델(예: Claude)은 [AGENTS.md](file:///D:/Design_AI_Harness/AGENTS.md)의 읽기 우선순위를 참고하여 현재 작업 상태와 락 상태를 검증합니다.
3. 작업이 끝나거나 컨텍스트 오버헤드가 발생하면 `scripts/print_handoff.ps1` 또는 `scripts/print_handoff.sh`를 구동하여 다음 AI가 이어받을 수 있는 Handoff 명세를 추출하여 활용합니다.
